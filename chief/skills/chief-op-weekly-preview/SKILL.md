# aireadylife-chief-weekly-preview

**Cadence:** Weekly (Monday morning)
**Produces:** Weekly agenda written to ~/Documents/AIReadyLife/vault/chief/00_current/week-YYYY-MM-DD.md

## What It Does

The weekly preview runs every Monday morning before the daily brief to give the user a full week-ahead picture before any single day's urgency can crowd out important but non-urgent work. It is the primary tool for intentional week design — not just reacting to today, but shaping what the entire week will accomplish.

The skill begins by calling `chief-flow-collect-domain-alerts` to pull all open loops across every installed plugin. From the full alert list, it filters for items that have an explicit due date or deadline falling within the next 7 days, or items marked 🔴 regardless of due date. These become the deadline calendar section of the weekly preview — a table showing each day of the upcoming week with the items due on that day, their source domain, and their urgency tier.

Next, the skill identifies 3-5 cross-domain priorities for the week. These are the most important things to accomplish this week — selected from 🔴 items without strict deadlines, high-priority 🟡 items, and any OKR-aligned objectives from vault/vision/ if the vision plugin is installed. Priority selection uses: urgency tier, how many weeks in a row the item has been on the list without movement (older unresolved items get priority over newer ones), and whether the item blocks other work.

Focus time recommendations are generated by reading vault/calendar/ if the calendar plugin is installed. The skill calculates total meeting hours per day of the coming week, identifies back-to-back meeting clusters (gaps under 30 minutes between events), and flags which days have at least one 90+ minute uninterrupted block available for deep work. Days with under 2 hours of total qualifying focus time are marked as focus-hostile and flagged for the user to consider meeting-free days or block protection.

The skill calls `chief-task-check-open-loops` to produce a backlog summary table showing total open-loop counts by domain and priority tier. This gives the user weekly visibility on whether the overall life backlog is shrinking or accumulating — a key system health signal.

The completed weekly agenda is written to vault/chief/00_current/week-YYYY-MM-DD.md (dated to the Monday of the current week).

- "weekly preview"
- "weekly agenda"
- "what's this week"

- "set up my week"

## Steps

1. Verify vault/chief/config.md exists; if missing, stop and prompt setup
2. Call `chief-flow-collect-domain-alerts` to gather all active open loops across installed plugins
3. Filter for items with due dates within the next 7 days; sort chronologically
4. Identify items marked 🔴 with no due date — add to priorities section
5. Read vault/calendar/ meeting load per day for the coming week (if calendar plugin installed)
6. Calculate qualifying focus time per day (blocks of 90+ uninterrupted minutes)
7. Flag days with <2 hours qualifying focus time as focus-hostile
8. Select 3-5 top priorities for the week from ranked cross-domain items
9. Call `chief-task-check-open-loops` for backlog count summary by domain and priority tier
10. Pass all inputs to `chief-flow-build-weekly-agenda` for document assembly
11. If vision plugin installed: read vault/vision/00_current/ for current-quarter OKRs and check alignment
12. Write completed weekly agenda to vault/chief/00_current/week-YYYY-MM-DD.md
13. Return formatted weekly preview to user as chat output

## Input

- ~/Documents/AIReadyLife/vault/*/open-loops.md (all installed plugin vaults)
- ~/Documents/AIReadyLife/vault/calendar/ (if calendar plugin installed, for meeting load data)
- ~/Documents/AIReadyLife/vault/vision/00_current/ (if vision plugin installed, for OKR alignment)
- ~/Documents/AIReadyLife/vault/chief/config.md

## Configuration

Required fields in vault/chief/config.md:
- `installed_plugins` — for auto-discovery override
- `focus_block_minimum_hours` — default: 1.5 (90 minutes); minimum block length to qualify as deep work time
- `weekly_priorities_override` — optional manual priorities that take precedence over auto-generated

## Error Handling

- **Calendar plugin not installed:** Show focus time section as "Calendar plugin not installed — install calendar plugin to enable focus time analysis."
- **No deadlines found for the week:** Show "No deadlines this week" for each day in the table — don't skip the table.
- **Vision plugin not installed:** Skip OKR alignment check silently; don't mention it in output.
- **Open loops exist but no due dates:** Items still appear in the backlog summary and priorities sections; just not in the deadline calendar table.

## Vault Paths
